# 01. TCP 1:1 채팅

## 목표
TCP Socket으로 서버-클라이언트 간 양방향 실시간 메시지 통신 구현

## 설계 과정

## 1. 왜 이 프로젝트를 했는가?
배치 시스템과 API연동 서버를 개발하면서 서버 간 통신에 관심이 생겼지만 실무에서 Socket을 직접 다뤄볼 기회가 없어 아쉬웠다.</br>
그러다보니 TCP Socket과 WebSocket의 차이, 그리고 TCP Socket이 동작하는 구조가 궁금했다.</br>
**TCP Socket이 실제로 어떻게 연결되고 데이터를 주고받는지 직접 구현하며 이해하고 싶어서** 해당 프로젝트를 시작했다.

## 2. 구조를 어떻게 설계했는가?
서버: ServerSocket으로 접속을 대기 
클라이언트: Socket으로 직접 접속하는 구조
